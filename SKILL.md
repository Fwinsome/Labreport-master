## 核心执行管线

```
素材接收 → 图片公式解析 → 内容填充 → 绘图拦截 → 最终输出
```

---

## 阶段一：素材接收

等待用户提供以下材料，按**顺序**接收：

1. 成品 PDF + 实验 PPT
2. 原始数据
3. 图片命名公式 + 实验图片

> **阻塞点**：必须收到全部三组材料后，方可进入阶段二。

---

## 阶段二：图片公式解析

**关键规则**：在处理任何图片之前，必须首先解析用户提供的"图片命名公式"，理解其结构逻辑。

解析内容包括：
- 命名格式中的变量含义（如 `V_1_1.png` 表示什么）
- 不同编号/后缀对应的实验组别
- 图片与报告章节（四/五/六）的对应关系

> **阻塞点**：必须向用户确认解析结果正确后，方可进入阶段三。

---

## 阶段三：内容填充

按以下顺序填充各章节内容：

1. **章节一（实验目的和要求）**：从 PPT 提炼
2. **章节二（实验内容和原理）**：从 PPT 提炼
3. **章节三（主要仪器设备）**：整理设备清单
4. **章节四（操作方法和实验步骤）**：从 PPT 提炼，含图片时按公式分组展示
5. **章节五（实验数据记录和处理）**：数据制表，有效数字处理，含图片时按公式分组展示
6. **章节六（实验结果与分析）**：数据绘图/分析，含图片时按公式分组展示
7. **章节七（思考与讨论）**：根据实验内容整理

**图片分层排版规则**：
- 同一组图片使用 `grid` 或 `subfigure` 横排
- 不同组别之间用**子标题（subheading）**区分
- 严格按照图片命名公式的逻辑分组，**禁止无序直排**

---

## 阶段四：绘图拦截（关键交互节点）

**触发条件**：在处理章节五或章节六时，遇到以下情况**立即停止** Typst 代码生成：

- 需要通过 Python 绘制额外分析图表
- 需要对数据做可视化（波特图、时域响应、幅频特性等）
- 任何涉及额外图像生成的场景

**拦截行为**：

1. **立即停止** Typst 代码生成
2. 向用户提问，明确以下内容：

```
【绘图确认请求】
在处理「[具体章节]」时，需要绘制以下图表：
- 图表类型：
- 数据来源：
- 样式要求：

请告诉我：
1. 图表的具体样式要求（如坐标轴标签、线型、图例等）
2. 是否需要额外的分析曲线
3. 图片的命名规则（用于分组展示）
```

**恢复条件**：必须收到用户的**明确绘图指示**后，方可继续生成相应代码。
